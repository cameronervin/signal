# Documentation Rules

Update docs after behavior changes.

| Change | Update |
| --- | --- |
| New endpoint | `backstage/api/endpoints.md` and API spec |
| New graph node or tool | `backstage/agents/tools.md` and agentic framework spec |
| Scoring change | `backstage/prd/02-technical-docs/01-signal/scoring.md` |
| Public API change | `backstage/prd/02-technical-docs/01-signal/integration-spec.md` |
| Data field change | `backstage/prd/02-technical-docs/01-signal/data-model.md` |
| Design token change | `backstage/design/README.md` and `.agents/style/design-tokens.md` |
| Setup change | `backstage/guides/setup.md` |

Docs should describe implemented behavior. Mark planned behavior as planned.
